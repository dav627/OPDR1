
小模型 Agentic 搜索后训练路线对比：GRPO 强化学习 vs 在策略蒸馏(OPD)          OPD · GRPO · veRL · RAG

项目简介：基于 ZeroSearch 模拟搜索框架（零真实 API 成本），在同一 Qwen2.5-3B 学生上系统对比 GRPO
与在策略蒸馏（OPD，7B 老师）两条后训练路线的能力上限与训练成本；单卡 A800 完成训练与评测全流程。

- OPD 最小改动实现：在 veRL 的 GRPO 代码路径上以单个 loss_mode 开关实现反向 KL 蒸馏——ref 模型
  替换为 7B 老师、KL 升为主损失（系数 1.0）、advantage 置零；state masking 保证仅在学生生成
  token 上蒸馏、检索返回内容不入损失；19 项单元测试校验损失逻辑，原 GRPO 路径可无损切回
- 首轮实验与问题定位：7 个 QA 基准上 OPD 较无训练基线 EM 0.208→0.327（+57%，多跳任务提升最
  大），但仅达 7B 老师（0.457）的 72%。经 wandb 训练曲线诊断该差距源于欠训练而非能力上限——lr
  warmup 占比设为 0.95 致 400 步训练全程处于 warmup 段、学习率从未到达峰值，分数曲线在截断点仍
  陡升；另通过代码追踪排除了"检索语料错配"的初始假设（该参数在训练管线中实为死代码）
- 公平评测方案设计：据诊断设计可复现的公平对比协议——修正学习率调度并延长训练步数、自训同条件
  GRPO 消除官方模型（14B 模拟器 + 更高算力）带来的混淆、4 模型×7 基准统一全量 test 集并以
  bootstrap 计算 95% 置信区间、绘制等算力预算下的 EM–GPU·h 曲线；撰写事前预注册文档固定各项预
  期，保证结论可被事后检验
- 阶段性结论：RL（GRPO）上限更高、可突破蒸馏天花板且多跳推理更强；OPD 无需 reward 工程、KL 单
  调下降训练更稳、性价比更优，但能力封顶于老师（反向 KL 的信息论上界）——为"有强老师选蒸馏、需
  突破上限选 RL"的小模型后训练选型提供实证依据
