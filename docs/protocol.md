## 0. 一句话结论（先给判断，后给证据）

在同一 Qwen2.5-3B 学生上对比 **GRPO 强化学习** 与 **OPD 在策略蒸馏（对 7B 老师做反向 KL）** 两条后训练路线。当前实测 OPD 只到老师 72%，但**四张 wandb 曲线证明 OPD 是在加速段中途被掐断的、远未收敛**；修正学习率调度并训到 800–1000 步后，预期 OPD 到老师 **85–95%（EM≈0.39–0.44）**，但**上限封顶于老师**（反向 KL 的信息论上界）。GRPO 上限更高、可突破老师，多跳任务尤甚。「谁更省钱」只能由等算力预算下的 EM–GPU·h 曲线裁决，本项目尚未产出该曲线。

## 1. 研究问题与假设

- **RQ1（能力上限）**：同一 3B 学生，RL 与蒸馏两条路线各自能到老师的百分之多少？
- **RQ2（性价比）**：等算力预算下，哪条路线更快到达目标水平？
- **RQ3（任务依赖）**：单跳 vs 多跳任务上两条路线的相对优劣是否不同？

**预注册假设**：

- **H1**：当前 OPD 的 0.327 是欠训练快照，不是极值。修正 `lr_warmup_steps_ratio=0.95` 这一病态调度、并加步后，OPD 显著上升到老师 85–95%。
- **H2**：无论如何加步/调参，OPD **不会超过老师** —— 反向 KL 本质是分布匹配，学生系统性超过老师在信息论上不成立。
- **H3**：token 级稠密监督使 OPD **早期（≤200 步）领先**，序列级稀疏 reward 的 GRPO 在中段交叉反超、终点略高；OPD 单步成本更低（可 `n_agent=1`），最终性价比由 EM–GPU·h 曲线决定。

- **指标**：Exact Match（EM，主指标）与 F1。

### 2.4 搜索 / 检索机制（关键细节）

- **`search_mode=simulate_sft`**：检索结果由**模拟器 LLM 生成**，不调用任何真实检索 API（零 API 成本）。模拟器 prompt 硬编码为 `"You are the Google search engine"`。
- **`search_engine` 是死代码（重要更正）**：`retriever.search_engine`（`google`/`wiki`）在 verl 训练/评测管线中**无任何 Python 代码消费**，仅独立脚本 `inference.py` 使用。因此**不存在** google/wiki 语料维度的分布偏移，也不存在「OPD 用 wiki 训、google 评」的口径偏差——早先文档的此归因已被代码追踪证伪（详见 §10 更正记录）。
- **噪声课程**：`start_threshold=0 → end_threshold=0.5`，随训练进度，模拟器生成的检索里「故意噪声文档」比例从 0 爬向 0.5（step 400 时约 40% 为噪声）。评测时传入 `start_threshold=0`，检索相对干净。
- **交互上限**：`max_turns=5`（最多 5 轮搜索），`topk=5`（每轮取 5 条）。

## 3. 方法与实现细节

### 3.1 两条路线在代码层的唯一实质差异

两条路线共用同一份 `main_ppo` 代码、同一份数据与模拟器、同一套 rollout/长度/batch 配置。**实质差异只有 ref 模型与 KL 系数两处**：

| 开关                   | GRPO 臂（`train_grpo.sh`）          | OPD 臂（`train_opd.sh`）                                |
| ---------------------- | ------------------------------------- | --------------------------------------------------------- |
| `actor.loss_mode`    | 默认（policy gradient / GRPO）        | `opd`（KL 为主损失，关闭 pg_loss）                      |
| `actor.use_kl_loss`  | `true`                              | `true`                                                  |
| `actor.kl_loss_coef` | `0.001`（小正则）                   | `1.0`（KL 即目标）                                      |
| `actor.kl_loss_type` | `low_var_kl`                        | `low_var_kl`                                            |
| `ref.model_path`     | **未覆盖**（默认=学生初始权重） | **`=7B 老师`**                                    |
| `adv_estimator`      | `grpo`                              | `grpo`（但 `loss_mode=opd` 下 advantage 项失效/置零） |

**解读**：GRPO 是「序列级 outcome reward（F1）+ 对学生自身初始策略的 0.001 弱 KL 正则」；OPD 是「把 ref 换成 7B 老师、KL 升为系数 1.0 的主损失、advantage 置零」，即在 on-policy rollout 上做**反向 KL 蒸馏**。这就是「在 GRPO 代码路径上以一个 `loss_mode` 开关最小改动实现 OPD」的含义。

### 3.2 检索 token 掩码（`state_masking=True`）

两臂都开 `state_masking=True`：损失/梯度只作用在**学生自己生成的 token** 上，模拟器返回的检索内容（observation）不计入 loss。这保证 OPD 的反向 KL 只在学生生成分布上蒸馏、GRPO 的策略梯度不被外部注入内容污染。

### 3.3 其他共享设置

`no_think_rl=false`（保留思考段）、`rollout.n_agent=5`（GRPO 的 group size；OPD 当前**沿用** 5，但蒸馏不需要 group rollout，见 §7 实验④）、`rollout.temperature=1`（训练采样温度）、`reward_function=f1`、`do_search=True`、`save_freq=200`、`test_freq=9999`（训练期**不做**中途评测——这是收敛性此前看不清的直接原因）、`total_epochs=10`、`total_training_steps` 由脚本参数给定。

### 3.4 评测协议（`run_phase_e.sh`）

- **`val_only=true` + `val_before_train=true`**：只做一次全量验证。
- **贪心、单样本、四模型一致**：`ray_trainer._validate` 内部固定 `meta_info={'do_sample': False, 'validate': True, ...}`，且验证路径**不按 `n_agent` 复制**。因此评测命令里 3B 的 `n_agent=5`/`temperature=1` 与 7B 的 `n_agent=1` **不影响验证解码**——验证一律 `do_sample=False` 贪心、每题一次生成。**解码协议是公平的**。
- **7B 评测的降显存处理**：7B 单卡塞不下时，`rollout_mem=0.25`、`val_batch=16`、`ref→3B baseline`、`ref_micro=2`（ref 仅用于记 log-prob，不影响 EM/F1 打分）。
- **当前题集不一致（待修）**：3B 系模型 **48 条/benchmark**，7B 老师 **195 条全量**——题集不同 + 单次运行 + 无 seed + 无置信区间。这是 §7 实验⑤要修的核心。

## 6. 收敛性诊断：OPD 远未收敛，是在加速段中途被掐断的

四张 wandb 曲线构成完整证据链：

- **`actor/lr`**：从 0 到 ~7.5e-7 的**纯直线**。`warmup_ratio=0.95 × 计划 500 步 = 475 步 warmup`，训练停在 **400 步**——整个 run 全程在 warmup 段内，**lr 连 1e-6 峰值都没到过**，无恒定期、无衰减期。
- **`critic/score/mean`（训练期 F1）**：前 ~250 步平在 0.25，约 **280 步起陡升**，400 步冲到 0.45–0.5，**截断点仍在爬、无平台**。而且这是在噪声课程加难（噪声比例 0→0.5）背景下涨出来的，真实学习斜率被低估。
- **`entropy_loss`** 从 ~0.95→~0.6、**`student_logp_mean`** 从 -0.9→-0.6，集中在最后 100–150 步。
- 三条曲线的「起飞点」整齐对在 lr 爬过 ~5e-7 处：**学习被 lr 点着，前 60% 步数基本空转。**

**一个要诚实标注的反面信号**：`actor/opd_loss`（对老师反向 KL）全程平在 1.4–1.5、无下降。两层含义：(1) 分布上学生离老师仍远，进一步支持「远未收敛」；(2) 该 KL 在 on-policy 分布漂移 + 噪声加难下**不是好进度计**——分数在涨、熵在降、KL 不动，说明学生先学到的是老师在关键 token 上的**行为**，整体 token 分布尚未贴合。**判断收敛以分数曲线为准，勿盯 KL。**

**结论**：0.327 / 72% 不是 OPD 的极值，甚至不是「接近饱和的欠拟合点」，而是加速段中途的一张快照。

## 7. 待补实验清单（公平评测方案，预注册）

> ①②③⑤ 是「完全公平」的必要条件，④ 决定能否谈性价比。每项给出「现状/设计/预期」。

### ① 同条件自训 GRPO（最关键缺口）

- **设计**：同学生初始权重、同数据、同 3B 模拟器、同 `search_engine`、同 batch/步数（500）/调度/`n_agent`/单卡 A800；两臂统一口径记 GPU·h。
- **预期**：自训 GRPO **低于**官方 0.444（14B 模拟器保真度更高 + 官方有效 batch 更大），平均 EM 约 **0.40–0.44（老师 88–96%）**；终点大概率仍 ≥ OPD，多跳优势定性保留，但「97% 追平老师」缩水到 ~90%。

### ② 查证老师训练时的模拟器规模（3B vs 14B）

- **背景**：原「三方分布对齐」诉求已证伪为死代码；此处改为查证老师蒸馏/评测所依赖的模拟器规模，排除保真度层面的隐性不对等。
- **新观察（削弱 OOD 假设）**：7B 老师**正是在这个 3B 模拟器上评出 0.457 的**，信号源在学生轨迹分布上可用。
- **预期**：主要作用是把配置钉死为单一可复现口径，**不预期带来明显性能变化**。原「分布对齐 +3~6 点」预期作废，提升空间归并到 ③。

### ③ lr / warmup 修正（两臂同规则、各取最优）

- **现状**：`lr=1e-6 + warmup_ratio=0.95`，等效 lr≈5e-7，病态调度对纯 KL 目标的 OPD 伤害大于 GRPO。
- **设计**：`warmup=0.1`，`lr∈{1e-6,2e-6,5e-6}` 小网格，两臂同规则、dev 上各取最优（tuned-arm 公平）。可与 ② 合并成一波重训。**因 ② 语料部分已死，本项是提升 OPD 的主杠杆。**
- **预期**：OPD 受益最大，单独此项 **+5~10 EM 点**；若几乎不动，则瓶颈锁定为分布偏移——此实验即报告 §5.3 两个归因的判别实验。②+③（+加步到 800）叠加后 OPD 预期到老师 **85–95%（EM≈0.39–0.44）**，但**不超过老师**（信息论上界，是结论而非缺陷）。

### ④ 等算力预算训练曲线（性价比结论的唯一合法来源）

- **设计**：两臂各训 500–800 步，每 100 步存档在 dev 评测，画 **EM–步数** 与 **EM–GPU·h** 双曲线；OPD 另跑 **`n_agent=1`** 版本（蒸馏不需 group rollout，现沿用 5 是被 GRPO 配置绑架的非自然设定）。
- **预期**：OPD 曲线平滑单调、**早期（≤200 步）领先**（token 级稠密信号 vs 序列级稀疏 reward），GRPO 在约 200–400 步交叉反超、终点高 0–3 点；OPD `n_agent=1` 单步成本降到 GRPO 的 **1/3–1/2**（rollout 是主开销），但每步更新 token 少、可能需更多步——**只有 EM–GPU·h 曲线能裁决性价比。**

### ⑤ 统一评测协议 + 显著性检验

- **现状**：3B 系 48 条 vs 老师 195 条、题集不同、单次运行、无 CI。n=48 时 EM 标准误约 **±0.07/基准**，2Wiki（+0.034）、MuSiQue（+0.016）的「反超」完全在噪声带内。
- **设计**：四模型跑**同一套全量 test**；paired bootstrap 出 95% CI（在现有 `*_record.json` 上**今天就能免费补算**）；从 train 切 dev 调参选点，test 只报一次；预算允许则两臂终点各跑 2–3 个 seed。
- **预期**：均值排序 `基线 < OPD < GRPO ≤ 老师` 稳（均值差 12 点远超均值 SE≈±0.03）；**「多跳反超老师」大概率退化为 CI 重叠的统计打平**，Bamboogle 老师领先（+30% 量级）仍显著。对外措辞从「反超」降为「持平/追平」。

### ⑥ 跨后端外部效度

- **设计**：全部模型再在 14B 模拟器 与 真实检索（e5+Wikipedia 的 Search-R1 标准配置，或 SerpAPI 小子集）各评一遍。
- **预期**：绝对分整体平移（ZeroSearch 论文称模拟≥真实），**相对排序不变则结论稳健**；若某模型换后端排名大变，说明过拟合了模拟器。

### ⑦（可选）SFT 蒸馏对照臂

老师轨迹上等 token 预算 SFT。预期 SFT 蒸馏 ≲ OPD，多跳/OOD 差距更大（多轮误差累积的 distribution shift），补齐「SFT 蒸馏 / OPD / RL」三方谱系。

### ⑧（可选）显式冷启动 + GRPO 组合臂

把原冷启动项目与本项目串成一条线，预期收敛步数少约 30%、小预算下终点略高。

## 8. 预期结果报告（公平化后，事前预期）

### 8.1 主表：公平化后的预期 EM

| 臂           | 现状 EM              | 公平化后预期 EM          | 相对老师 | 依赖实验 |
| ------------ | -------------------- | ------------------------ | -------- | -------- |
| 基线 3B      | 0.208（已测）        | 不变                     | ~45%     | —       |
| OPD 3B       | 0.327                | **0.39–0.44**     | 85–95%  | ②③④   |
| 自训 GRPO 3B | —（现用官方 0.444） | **0.40–0.44**     | 88–96%  | ①       |
| 7B 老师      | 0.457（已测）        | 统一题集后 ±0.02 内浮动 | 100%     | ⑤       |

### 8.2 预期曲线

- **EM–步数**：OPD 平滑单调、≤200 步领先；GRPO 在 200–400 步交叉反超、终点高 0–3 点。
- **EM–GPU·h**：OPD（尤其 `n_agent=1`）在小预算区间性价比占优（单步成本约 GRPO 的 1/2–1/3）；GRPO 在充足预算下终点更高。交叉点位置是性价比结论的关键。
- **中途 checkpoint 预期**：`EM(step 200) ≈ 0.24–0.27`（起飞发生在 ~280 步之后），`EM(step 400)=0.327`（已测）。若 200→400 斜率仍陡，「未饱和」由推断变实测。

### 8.3 预期显著性与措辞

- 均值层面排序 `基线 < OPD < GRPO ≤ 老师` 显著稳定。
- 多跳「反超老师」（2Wiki +0.034、MuSiQue +0.016）**预期退化为 CI 重叠的统计打平**；Bamboogle 老师领先显著。
- 因此对外表述：OPD/GRPO 在多跳上「**追平/持平**」老师，而非「反超」。

### 8.4 预期核心结论（RQ 的预期答案）

- **RQ1**：GRPO 能榨到老师 ~90%+ 并在部分多跳打平/略超；OPD 到老师 85–95% 但**封顶于老师**。
- **RQ2**：小预算下 OPD 性价比更优（稠密监督、无 rollout group、无 reward 工程、KL 单调下降训练稳）；这一点在**本项目须由 ④ 的 EM–GPU·h 曲线实证**，当前仅为机制与文献层面判断。
- **RQ3**：RL 在复杂多跳推理上优势更明显；蒸馏在单跳上追得更紧。

### 8.5 OPD 性价比：结论、前提与边界（机制层）

**结论成立，准确版本**：*在「已有更强老师、且能拿到其逐 token logprob」的前提下，OPD 是把小模型推到接近老师水平的最省钱路线。* 省钱主要省的不是单步开销，而是**到达目标水平所需的 episode 数**（老师逐 token 稠密监督 vs RL 单标量 reward）。文献一致：Qwen3 小尺寸用蒸馏替代 RL 约 1/10 GPU 时且更好；DeepSeek-R1 对 32B 直接 RL 不如从 R1 蒸馏；Thinking Machines 报告 on-policy 蒸馏以约一个数量级更低算力达 RL 水平。

- **边界一：上限=老师**（反向 KL 是分布匹配，学生系统性超越老师信息论上不成立；要突破上限或无老师时，RL 是唯一路线）。
- **边界二：强依赖匹配度**（tokenizer 同系、老师在学生实际会走到的状态分布上可信；本项目老师在同一 3B 模拟器上的 0.457 说明此条满足）。

## 9. 执行计划与优先级

立即可做的两个**低成本验证**（比任何论证都硬，先做）：

1. **评 step-200 checkpoint**：`save_freq=200`，step 200 checkpoint 大概率仍在盘上。约 1 小时评一遍即得 `0.208(step0) → EM(200) → 0.327(step400)` 三点。
2. **出分数–步数曲线**：下一轮把 `test_freq` 从 9999 改为 50–100（用小 dev 子集，**勿用 test** 免隐性过拟合）。

整体优先级：

```
⑤ 的 bootstrap（零成本，先做）
→ §9 step-200 评测（约 1h，把「未收敛」坐实）
→ ③ lr/warmup 修正（判别性最强）
→ ②③ 合并重训 OPD（warmup=0.1, lr 网格, 训到 800–1000 步）
→ ① 自训同条件 GRPO（顺带产出 ④ 的曲线）
→ ⑥ 跨后端外部效度
```
