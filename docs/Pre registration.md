## 2. 收敛性分析：OPD 远未收敛，是在加速段中途被掐断的

**结论：0.327 / 达老师 72% 不是 OPD 的极值，甚至不是一个「接近饱和的欠拟合点」，而是加速段中途的一张快照。**

四张 wandb 曲线放在一起构成完整证据链：

- **`actor/lr`**：一条从 0 到 ~7.5e-7 的纯直线。`warmup_ratio=0.95 × 计划 500 步 = 475 步 warmup`，而训练停在 **400 步**——意味着整个 run 从头到尾都在 warmup 段内，**lr 连 1e-6 的峰值都从未到达过**，更谈不上恒定期或衰减期。
- **`critic/score/mean`**（训练期 F1；OPD 不用它做损失，但照常记录）：前 ~250 步平在 0.25 附近，约 **280 步起陡升**，到 400 步已冲到 0.45–0.5，**截断点上曲线仍在爬升、没有任何平台迹象**。而且这是在**噪声课程逐步加难**的背景下涨出来的——`gt_threshold` 随步数从 0 爬向 0.5，step 400 时约 40% 的检索是故意生成的噪声文档——所以真实学习斜率只会被这条曲线低估。
- **`entropy_loss`** 从 ~0.95 跌到 ~0.6，**`student_logp_mean`** 从 -0.9 抬到 -0.6，全部集中在最后 100–150 步。
- 三组曲线的「起飞点」整齐地对在 lr 爬过 ~5e-7 的位置：**学习是被 lr 点着的，前 60% 的步数模型基本在空转。**

**一个要诚实标注的反面信号**：`actor/opd_loss`（对老师的反向 KL）全程平在 1.4–1.5，**没有下降趋势**。两层含义：

1. 分布意义上学生离老师还很远——反而进一步支持「远未收敛」。
2. 这个 KL 在 on-policy 数据分布持续漂移 + 噪声课程加难的设定下**不是好的进度计**。分数在涨、熵在降、KL 不动，说明学生先学会的是老师在任务关键 token 上的**行为**，整体 token 分布还没贴上去。**下一轮判断收敛请以分数曲线为准，别盯 KL。**

## 3. 公平评测实验清单（①–⑥ 必补，⑦⑧ 可选）

### ① 同条件自训 GRPO（最关键的缺口）

- **现状**：GRPO 臂用的是官方模型（203 步、14B 模拟器、官方多卡预算），OPD 臂是自训的（400 步、3B 模拟器、单卡 batch 32）——「路线差异」和「训练条件差异」完全混淆。报告 §3.4 自己也承认不能下性价比结论。
- **设计**：同学生初始权重、同数据、同 3B 模拟器、同 `search_engine`、同 batch/步数（500）/调度/`n_agent`/单卡 A800；两臂用统一口径记 GPU·h。
- **预期**：自训 GRPO 会**低于**官方 0.444（14B 模拟器保真度更高 + 官方有效 batch 更大），平均 EM 约 **0.40–0.44（老师的 88–96%）**；终点大概率仍 ≥ OPD，多跳优势定性保留，但「97% 追平老师」会缩水到 90% 上下。

### ② 查证老师训练时的模拟器规模（3B vs 14B）

- **背景**：原「分布对齐」诉求已被证伪为死代码（见 §1）。此处改为查证老师蒸馏/评测所依赖的模拟器规模，排除保真度层面的隐性不对等。
- **预期**：由于老师在同一 3B 模拟器上已能评出 0.457，信号源可用，此项主要作用是把评测配置钉死为可复现的单一口径，而非预期带来明显性能变化。原「+3~6 点」预期作废。

### ③ lr / warmup 修正（两臂同规则、各取最优）

- **现状**：`lr=1e-6 + warmup_ratio=0.95`，400 步里约 380 步在爬坡，等效 lr≈5e-7；病态调度对纯 KL 目标的 OPD 伤害大于 GRPO，「共享同一个坏调度」≠ 公平。
- **设计**：`warmup` 降到 0.1，`lr ∈ {1e-6, 2e-6, 5e-6}` 小网格，两臂同规则、在 dev 上各取最优（tuned-arm 公平）。可与 ② 合并成一波重训。**由于 ② 的语料部分已是死代码，本项现在是提升 OPD 的主杠杆。**
- **预期**：OPD 受益最大，单独此项 **+5~10 个 EM 点**；若几乎不动，则锁定瓶颈为分布偏移——这个实验本身就是报告 §5.3 两个归因的判别实验。②+③（+加步到 800）叠加后，OPD 预期到老师的 **85–95%（EM≈0.39–0.44）**，但**不会超过老师**——反向 KL 的信息论上界，这是要呈现的结论而非缺陷。

### ④ 等算力预算训练曲线（性价比结论的唯一合法来源）

- **设计**：两臂各训 500–800 步，每 100 步存档在 dev 上评测，画 **EM–步数** 和 **EM–GPU·h** 双曲线；OPD 另跑一个 `n_agent=1` 版本——蒸馏不需要 group rollout，现在沿用 `n_agent=5` 是被 GRPO 配置绑架的非自然设定。
- **预期**：OPD 曲线平滑单调、**早期（≤200 步）领先**（token 级稠密信号 vs 序列级稀疏 reward），GRPO 在约 200–400 步交叉反超、终点高 0–3 点；OPD `n_agent=1` 单步成本降到 GRPO 的 **1/3–1/2**（rollout 是主开销），但每步更新 token 少、可能需要更多步——所以**只有 EM–GPU·h 曲线能裁决「性价比」**。

### ⑤ 统一评测协议 + 显著性检验

- **现状**：3B 系 48 条/基准 vs 老师 195 条全量，题集都不同；单次运行、无 seed、无置信区间。n=48 时 EM 的标准误约 **±0.07/基准**，2Wiki（+0.034）、MuSiQue（+0.016）的「反超」完全在噪声带内；另外直接在 test 上看结果选 checkpoint 有隐性过拟合。
- **设计**：四个模型跑同一套全量 test 题集；paired bootstrap 出 95% CI（在现有 `*_record.json` 上今天就能免费补算）；从 train 切 dev 用于调参选点，test 只报一次；预算允许则两臂终点各跑 2–3 个训练 seed。
- **预期**：均值层面排序 `基线 < OPD < GRPO ≤ 老师` 稳（均值差 12 点远超均值 SE≈±0.03）；**「多跳反超老师」大概率退化为 CI 重叠的统计打平**，Bamboogle 老师领先（+30% 量级）仍显著。对外措辞相应从「反超」降为「持平/追平」。

### ⑥ 跨后端外部效度

- **设计**：全部模型再在 14B 模拟器和真实检索（e5+Wikipedia 的 Search-R1 标准配置，或 SerpAPI 小子集）各评一遍。
- **预期**：绝对分整体平移（ZeroSearch 论文称模拟≥真实），**相对排序不变则结论稳健**；若某模型换后端排名大变，说明过拟合了模拟器。

### ⑦（可选）SFT 蒸馏对照臂

老师轨迹上等 token 预算 SFT。预期 SFT 蒸馏 ≲ OPD，多跳/OOD 差距更大（多轮误差累积的 distribution shift），补齐「SFT 蒸馏 / OPD / RL」三方谱系。

### ⑧（可选）显式冷启动 + GRPO 组合臂

把原项目（冷启动）和本项目串成一条线，预期收敛步数少约 30%（与已有结论一致）、小预算下终点略高。

## 4. 公平化后的预期汇总

| 臂                 | 现状 EM              | 公平化后预期 EM          | 相对老师 |
| ------------------ | -------------------- | ------------------------ | -------- |
| 基线 3B            | 0.208（已测）        | 不变                     | ~45%     |
| OPD 3B（②③④后） | 0.327                | 0.39–0.44               | 85–95%  |
| 自训 GRPO 3B（①） | —（现用官方 0.444） | 0.40–0.44               | 88–96%  |
| 7B 老师            | 0.457（已测）        | 统一题集后 ±0.02 内浮动 | 100%     |

**置信度说明**：中心预期不变、置信度较上一轮**上调**，三个理由——(1) 欠训练的证据现在是直接可见的（§2），不再是推断；(2) 最强的替代解释（语料错配）已被排除成死代码；(3) 老师在同一 3B 模拟器上的 0.457 证明蒸馏信号源在该分布上成立。但有一句不因「没收敛」而改变：**饱和点仍不会超过老师**——反向 KL 的上界是信息论性质的，加步数改变的是离天花板多近，不是天花板本身。

## 5. OPD 性价比：结论与边界

**结论成立，但有一个硬前提和两个边界。**准确版本是：*在「已经有一个更强的老师、且能拿到它的逐 token logprob」的前提下，OPD 是把小模型推到接近老师水平的最省钱路线。*

**机制上省钱的原因**：老师给的是逐 token 的稠密监督，一条轨迹几百上千 bit 的信号，而 RL 一条轨迹只有一个标量 reward；OPD 不需要 group rollout（n=1 即可）、不需要 critic、不需要 reward 工程，也没有 reward hacking，KL 单调下降训练很稳。所以它省的主要不是单步开销，而是**到达目标水平所需的 episode 数**。公开证据一致：Qwen3 报告里小尺寸模型用蒸馏替代 RL，约 1/10 GPU 时且效果更好；DeepSeek-R1 论文里对 32B 直接做 RL 明显不如从 R1 蒸馏（那是离策略 SFT 蒸馏，OPD 在此基础上进一步消掉了分布偏移）；Thinking Machines 在数学推理上报告 on-policy 蒸馏以约一个数量级更低的算力达到 RL 水平。

**边界一：效果上限 = 老师。**反向 KL 本质是分布匹配，学生系统性超过老师在信息论上不成立。所以「较有效果」准确说是「高效地逼近老师」，不是绝对效果更强——要突破老师、或压根没有老师（前沿能力、老师没见过的工具生态/私有域），RL 是唯一路线。行业标准分工也是这个逻辑：顶层模型用 RL 往上顶，部署尺寸靠蒸馏往下铺。

**边界二：强依赖匹配度。**tokenizer 要同系（逐 token KL 才对得齐），老师要在学生实际会走到的状态分布上可信。本项目里老师在同一 3B 模拟器上的 0.457 说明这一条是满足的。

**一句重要提醒**：这个性价比结论目前是**文献共识，还不是本项目的实测结论**——OPD 臂当前只到老师 72%（且已判定为欠训练，见 §2），GRPO 臂又没有同条件成本数据。只有等实验 ④ 的 EM–GPU·h 曲线出来，才能把它换成本项目自己的数字。

## 6. 立即可做的低成本验证

比任何论证都硬的两件事，建议先做：

1. **评 step-200 checkpoint**：`train_opd.sh` 里 `save_freq=200`，step 200 的 checkpoint 大概率还在盘上。花约 1 小时评一遍，就有 `0.208（step 0）→ EM(200) → 0.327（step 400)` 三个点。预期 EM(200) 在 **0.24–0.27**（起飞发生在 ~280 步之后）；如果 200→400 段斜率仍很陡，「未饱和」就从推断变成实测。
2. **出分数–步数曲线**：下一轮训练把 `test_freq` 从 9999 改成 50–100（用小 dev 子集，**别用 test** 免得隐性过拟合），直接产出分数–步数曲线，饱和点不用再猜。
